Definición

"La arquitectura de computadores se refiere a la estructura y organización de los componentes hardware de un sistema computacional, incluyendo la forma en que se interconectan y funcionan juntos."

Niveles de Memoria

Registros

Más rápidos, menor capacidad, mayor coste

  • Velocidad: Muy alta
  • Capacidad: 32-64 bytes
  • Coste: Muy alto

Cache L1

Memoria cache de primer nivel

  • Velocidad: Alta
  • Capacidad: 32-64 KB
  • Coste: Alto

Cache L2

Memoria cache de segundo nivel

  • Velocidad: Media-alta
  • Capacidad: 256 KB - 1 MB
  • Coste: Medio

RAM

Memoria principal del sistema

  • Velocidad: Media
  • Capacidad: 8-64 GB
  • Coste: Bajo

Disco Duro

Almacenamiento masivo

  • Velocidad: Baja
  • Capacidad: 1-10 TB
  • Coste: Muy bajo

Registros

Tipos de Buses

Bus de Datos

Transfiere datos entre componentes

  • Ancho: 32, 64, 128 bits
  • Velocidad: Alta
  • Bidireccional

Bus de Direcciones

Especifica ubicaciones de memoria

  • Ancho: 32, 64 bits
  • Unidireccional
  • Desde CPU hacia memoria

Bus de Control

Señales de control y sincronización

  • Señales de lectura/escritura
  • Interrupciones
  • Reloj del sistema

Tipos de Arquitecturas

Diferentes enfoques en el diseño de sistemas computacionales

Clasificación de Arquitecturas

Arquitectura von Neumann

Arquitectura clásica con memoria compartida

  • Memoria unificada para datos e instrucciones
  • Ejecución secuencial de instrucciones
  • Cuello de botella en el bus de memoria

Arquitectura Harvard

Memorias separadas para datos e instrucciones

  • Buses independientes
  • Mayor paralelismo
  • Mejor rendimiento

Arquitectura RISC

Conjunto de instrucciones reducido

  • Instrucciones simples y rápidas
  • Pipeline eficiente
  • Menor consumo energético

Arquitectura CISC

Conjunto de instrucciones complejo

  • Instrucciones potentes
  • Menor número de instrucciones
  • Mayor complejidad del hardware
