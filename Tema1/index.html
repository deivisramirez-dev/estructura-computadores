Niveles de Descripción del Computador

Distintos modelos de niveles desarrollados a lo largo de la evolución de los computadores:

Niveles Conceptuales de Blaaw (1970)

Estructura conceptual que organiza los diferentes niveles de abstracción en el diseño de computadores.

Niveles Estructurales de Bell y Newell (1971)

Modelo que describe la organización estructural de los componentes del computador en diferentes niveles.

Niveles de Interpretación de Levy (1978)

Sistema de niveles que describe cómo se interpretan y ejecutan las instrucciones en diferentes capas.

Niveles Funcionales de Tanenbaum (1986)

Arquitectura funcional que organiza los niveles según las funciones que realizan en el sistema.

Conceptos para Interpretar el Modelo

Estructura

Los componentes que un nivel inferior utiliza para dar soporte a un nivel superior.

Ejemplo: Transistores → Compuertas → ALU → Procesador
Organización

Los flujos de control y de datos entre los componentes.

Ejemplo: Buses, señales de control, caminos de datos
Descripción Funcional

Las características de comportamiento de un elemento, que sirven de base para un nivel superior.

Ejemplo: Comportamiento de una ALU para el nivel de sistema

Tipos de Paralelismo en Flynn

Paralelismo de Datos

La misma función o instrucción se ejecuta en paralelo, pero en cada ejecución se aplica sobre un conjunto de datos distinto.

• Común en arquitecturas SIMD • Procesamiento vectorial • GPU y procesadores vectoriales

Paralelismo Funcional

Varias funciones o tareas (iguales o distintas) se ejecutan en paralelo.

• Característico de arquitecturas MIMD • Multiprocesadores y multicomputadores • Mayor flexibilidad

Niveles de Paralelismo Funcional

El paralelismo funcional se clasifica según la "granularidad" o el tamaño de las tareas que se ejecutan en paralelo:

Nivel de Instrucción Granularidad Fina

Paralelismo a nivel de instrucciones individuales (ILP - Instruction Level Parallelism)

• Pipeline de instrucciones • Ejecución superescalar • Predicción de saltos
Nivel de Bucle o Hebra Granularidad Fina-Media

Ejecución paralela de bucles o hilos (threads)

• Paralelización de bucles • Multithreading • Hyperthreading
Nivel de Procedimiento Granularidad Media

Ejecución paralela de procesos o procedimientos

• Multiprocesamiento • Procesos concurrentes • Comunicación entre procesos
Nivel de Programa Granularidad Gruesa

Ejecución paralela de programas completos

• Sistemas distribuidos • Clusters de computadores • Computación en la nube

Tipos de Arquitectura

SISD

Single Instruction, Single Data

  • Un procesador, un flujo de datos
  • Arquitectura tradicional secuencial
  • Ejemplo: Procesador monocore clásico

SIMD

Single Instruction, Multiple Data

  • Una instrucción, múltiples datos
  • Procesamiento vectorial
  • Ejemplo: GPU, procesadores vectoriales

MISD

Multiple Instruction, Single Data

  • Múltiples instrucciones, un dato
  • Arquitectura poco común
  • Ejemplo: Sistemas de redundancia

MIMD

Multiple Instruction, Multiple Data

  • Múltiples instrucciones, múltiples datos
  • Arquitectura más flexible
  • Ejemplo: Multicores, clusters

Evaluación de Prestaciones

Factores tecnológicos y métodos de evaluación del rendimiento

Factores Tecnológicos

Densidad de Integración

Número de transistores por unidad de área

• Ley de Moore: duplicación cada 18-24 meses • Mayor complejidad en menor espacio • Mejoras en eficiencia energética

Capacidad de Almacenamiento

Cantidad de datos que puede almacenar

• Memorias semiconductoras más grandes • Mejores tecnologías de almacenamiento • Mayor velocidad de acceso

Frecuencia de Funcionamiento

Velocidad de reloj del procesador

• Mayor frecuencia máxima de funcionamiento • Mejoras en arquitectura de pipeline • Optimización de ciclos por instrucción

Computación Científica

Necesidades de procesamiento intensivo que impulsan el desarrollo de nuevas arquitecturas.

• Simulaciones complejas • Análisis de datos masivos • Modelado matemático

Comunicación Global

El desarrollo de Internet y las redes globales condiciona la evolución arquitectónica.

• Procesamiento de redes • Servidores web y aplicaciones • Comunicación en tiempo real

Dispositivos Móviles
