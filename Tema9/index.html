OpenMP (Memoria Compartida)

¿Qué es OpenMP?

OpenMP (Open Multi-Processing) es un estándar de programación paralela para sistemas de memoria compartida. Permite paralelizar programas de forma relativamente sencilla usando directivas del compilador.

🔧 Características Principales:
  • Directivas del compilador: Instrucciones especiales que le dicen al compilador cómo paralelizar el código
  • Paralelización automática: El compilador se encarga de crear y gestionar los hilos
  • Memoria compartida: Todos los hilos pueden acceder a la misma memoria
  • Fácil de usar: Solo necesitas agregar directivas a tu código existente
🏗️ Arquitectura:

OpenMP funciona en sistemas donde múltiples procesadores comparten la misma memoria RAM. Es ideal para:

  • Servidores con múltiples núcleos
  • Estaciones de trabajo de alto rendimiento
  • Supercomputadores con memoria compartida

Directivas Principales:

  • #pragma omp parallel: Crea un equipo de hilos
  • #pragma omp parallel for: Paraleliza un bucle
  • #pragma omp critical: Sección crítica (solo un hilo a la vez)
  • #pragma omp reduction: Operaciones de reducción (suma, producto, etc.)

Ejemplo 1: Suma de Elementos de un Vector

📋 Enunciado:

Dado un vector de N elementos, calcular la suma de todos sus elementos utilizando OpenMP. El programa debe mostrar el tiempo de ejecución y el resultado final.

#include <omp.h>
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

#define N 1000000

int main() {
  int *vector = (int*)malloc(N * sizeof(int));
  long long suma = 0;
  double start_time, end_time;

  // Inicializar vector con valores aleatorios
  for(int i = 0; i < N; i++) {
    vector[i] = rand() % 100;
  }

  start_time = omp_get_wtime();
  
  // Paralelización con OpenMP
  #pragma omp parallel for reduction(+:suma)
  for(int i = 0; i < N; i++) {
    suma += vector[i];
  }
  
  end_time = omp_get_wtime();
  
  printf("Suma total: %lld\n", suma);
  printf("Tiempo de ejecución: %f segundos\n", end_time - start_time);
  printf("Hilos utilizados: %d\n", omp_get_max_threads());
  
  free(vector);
  return 0;
}
🔍 Explicación Detallada:
  • #pragma omp parallel for: Crea múltiples hilos y distribuye las iteraciones del bucle entre ellos
  • reduction(+:suma): Cada hilo mantiene una copia local de 'suma', al final se suman todas las copias
  • omp_get_wtime(): Función de OpenMP para medir tiempo de ejecución con alta precisión
  • omp_get_max_threads(): Obtiene el número máximo de hilos disponibles
  • Ventaja: El bucle se ejecuta en paralelo, reduciendo significativamente el tiempo de ejecución

Ejemplo 2: Multiplicación de Matrices

📋 Enunciado:

Implementar la multiplicación de dos matrices cuadradas de NxN utilizando OpenMP. Comparar el rendimiento con diferentes números de hilos.

#include <omp.h>
#include <stdio.h>
#include <stdlib.h>

#define N 512

void matrix_multiply(double **A, double **B, double **C) {
  #pragma omp parallel for
  for(int i = 0; i < N; i++) {
    for(int j = 0; j < N; j++) {
      C[i][j] = 0.0;
      for(int k = 0; k < N; k++) {
        C[i][j] += A[i][k] * B[k][j];
      }
    }
  }
}

int main() {
  // Asignar memoria para matrices
  double **A = (double**)malloc(N * sizeof(double*));
  double **B = (double**)malloc(N * sizeof(double*));
  double **C = (double**)malloc(N * sizeof(double*));
  
  for(int i = 0; i < N; i++) {
    A[i] = (double*)malloc(N * sizeof(double));
    B[i] = (double*)malloc(N * sizeof(double));
    C[i] = (double*)malloc(N * sizeof(double));
  }
  
  // Inicializar matrices
  for(int i = 0; i < N; i++) {
    for(int j = 0; j < N; j++) {
      A[i][j] = 1.0;
      B[i][j] = 1.0;
    }
  }
  
  double start = omp_get_wtime();
  matrix_multiply(A, B, C);
  double end = omp_get_wtime();
  
  printf("Tiempo de ejecución: %f segundos\n", end - start);
  printf("Hilos utilizados: %d\n", omp_get_num_threads());
  
  // Liberar memoria
  for(int i = 0; i < N; i++) {
    free(A[i]); free(B[i]); free(C[i]);
  }
  free(A); free(B); free(C);
  return 0;
}
🔍 Explicación Detallada:
  • Paralelización del bucle externo: Solo el bucle 'i' se paraleliza para evitar condiciones de carrera
  • Independencia de datos: Cada hilo calcula filas diferentes de la matriz resultado
  • Acceso a memoria: Cada hilo accede a diferentes filas, minimizando conflictos de caché
  • Escalabilidad: El rendimiento mejora proporcionalmente al número de hilos (hasta cierto punto)
  • Complejidad: O(N³) con paralelización O(N²) en el bucle externo

MPI (Memoria Distribuida)

¿Qué es MPI?

MPI (Message Passing Interface) es un estándar de programación paralela para sistemas de memoria distribuida. Permite que múltiples procesos independientes se comuniquen enviándose mensajes.

🔧 Características Principales:
  • Comunicación por mensajes: Los procesos se comunican enviándose datos explícitamente
  • Memoria distribuida: Cada proceso tiene su propia memoria privada
  • Escalabilidad horizontal: Puede ejecutarse en múltiples máquinas conectadas por red
  • Independencia de procesos: Cada proceso es independiente y puede fallar sin afectar a otros
🏗️ Arquitectura:

MPI funciona en sistemas donde cada procesador tiene su propia memoria. Es ideal para:

  • Clusters de computadoras
  • Supercomputadores distribuidos
  • Grid computing (computación en red)
  • Cloud computing distribuido

Funciones Principales:

  • MPI_Init/MPI_Finalize: Inicializar y finalizar MPI
  • MPI_Send/MPI_Recv: Envío y recepción de mensajes
  • MPI_Bcast: Difusión (broadcast) de datos
  • MPI_Reduce: Reducción de datos entre procesos
  • MPI_Gather/MPI_Scatter: Recolección y distribución de datos

Ejemplo 1: Suma Distribuida de un Vector

📋 Enunciado:

Calcular la suma de un vector de N elementos distribuido entre P procesos usando MPI. El proceso 0 debe mostrar el resultado final.

#include <mpi.h>
#include <stdio.h>
#include <stdlib.h>

#define N 1000000

int main(int argc, char *argv[]) {
  int rank, size, i;
  int *vector, *local_vector;
  int local_size, local_sum = 0, global_sum = 0;
  double start_time, end_time;

  MPI_Init(&argc, &argv);
  MPI_Comm_rank(MPI_COMM_WORLD, &rank);
  MPI_Comm_size(MPI_COMM_WORLD, &size);

  local_size = N / size;
  local_vector = (int*)malloc(local_size * sizeof(int));

  if(rank == 0) {
    vector = (int*)malloc(N * sizeof(int));
    for(i = 0; i < N; i++) {
      vector[i] = rand() % 100;
    }
    start_time = MPI_Wtime();
  }

  // Distribuir datos
  MPI_Scatter(vector, local_size, MPI_INT, local_vector, local_size, MPI_INT, 0, MPI_COMM_WORLD);

  // Calcular suma local
  for(i = 0; i < local_size; i++) {
    local_sum += local_vector[i];
  }

  // Reducir suma global
  MPI_Reduce(&local_sum, &global_sum, 1, MPI_INT, MPI_SUM, 0, MPI_COMM_WORLD);

  if(rank == 0) {
    end_time = MPI_Wtime();
    printf("Suma total: %d\n", global_sum);
    printf("Tiempo de ejecución: %f segundos\n", end_time - start_time);
    printf("Procesos utilizados: %d\n", size);
    free(vector);
  }

  free(local_vector);
  MPI_Finalize();
  return 0;
}
🔍 Explicación Detallada:
  • MPI_Scatter: Distribuye el vector completo desde el proceso 0 a todos los procesos
  • MPI_Reduce: Suma todas las sumas locales y envía el resultado al proceso 0
  • MPI_Wtime(): Función de MPI para medir tiempo de ejecución
  • Distribución de trabajo: Cada proceso calcula la suma de su porción del vector
  • Ventaja: Escalabilidad horizontal - más procesos = mejor rendimiento

Ejemplo 2: Multiplicación de Matrices Distribuida

📋 Enunciado:

Implementar la multiplicación de matrices A×B=C distribuyendo las filas de A entre los procesos. Cada proceso calcula sus filas correspondientes de C.

#include <mpi.h>
#include <stdio.h>
#include <stdlib.h>

#define N 512

int main(int argc, char *argv[]) {
  int rank, size, i, j, k;
  int rows_per_proc, start_row, end_row;
  double **A, **B, **C;
  double *local_A, *local_C;
  double start_time, end_time;

  MPI_Init(&argc, &argv);
  MPI_Comm_rank(MPI_COMM_WORLD, &rank);
  MPI_Comm_size(MPI_COMM_WORLD, &size);

  rows_per_proc = N / size;
  start_row = rank * rows_per_proc;
  end_row = start_row + rows_per_proc;

  if(rank == 0) {
    // Inicializar matrices completas en proceso 0
    A = (double**)malloc(N * sizeof(double*));
    B = (double**)malloc(N * sizeof(double*));
    C = (double**)malloc(N * sizeof(double*));
    
    for(i = 0; i < N; i++) {
      A[i] = (double*)malloc(N * sizeof(double));
      B[i] = (double*)malloc(N * sizeof(double));
      C[i] = (double*)malloc(N * sizeof(double));
    }
    
    // Inicializar con valores
    for(i = 0; i < N; i++) {
      for(j = 0; j < N; j++) {
        A[i][j] = 1.0;
        B[i][j] = 1.0;
      }
    }
    start_time = MPI_Wtime();
  }

  // Distribuir filas de A
  local_A = (double*)malloc(rows_per_proc * N * sizeof(double));
  local_C = (double*)malloc(rows_per_proc * N * sizeof(double));
  
  MPI_Scatter(A[0], rows_per_proc * N, MPI_DOUBLE, local_A, rows_per_proc * N, MPI_DOUBLE, 0, MPI_COMM_WORLD);
  
  // Broadcast matriz B a todos los procesos
  if(rank != 0) B = (double**)malloc(N * sizeof(double*));
  if(rank != 0) {
    for(i = 0; i < N; i++) {
      B[i] = (double*)malloc(N * sizeof(double));
    }
  }
  MPI_Bcast(B[0], N * N, MPI_DOUBLE, 0, MPI_COMM_WORLD);

  // Multiplicación local
  for(i = 0; i < rows_per_proc; i++) {
    for(j = 0; j < N; j++) {
      local_C[i * N + j] = 0.0;
      for(k = 0; k < N; k++) {
        local_C[i * N + j] += local_A[i * N + k] * B[k][j];
      }
    }
  }

  // Recopilar resultados
  MPI_Gather(local_C, rows_per_proc * N, MPI_DOUBLE, C[0], rows_per_proc * N, MPI_DOUBLE, 0, MPI_COMM_WORLD);

  if(rank == 0) {
    end_time = MPI_Wtime();
    printf("Tiempo de ejecución: %f segundos\n", end_time - start_time);
    printf("Procesos utilizados: %d\n", size);
  }

  free(local_A); free(local_C);
  if(rank != 0) {
    for(i = 0; i < N; i++) free(B[i]);
    free(B);
  }
  MPI_Finalize();
  return 0;
}
🔍 Explicación Detallada:
  • MPI_Scatter: Distribuye filas de A entre procesos
  • MPI_Bcast: Envía matriz B completa a todos los procesos
  • MPI_Gather: Recopila resultados de todos los procesos
  • Distribución por filas: Cada proceso calcula filas específicas de C
  • Comunicación colectiva: Uso eficiente de operaciones MPI para distribución de datos

📊 Resumen de Diferencias:

OpenMP

Mejor para: Programas que necesitan compartir datos frecuentemente entre hilos, algoritmos con dependencias complejas, desarrollo rápido de prototipos.

Limitaciones: Limitado a una sola máquina, número limitado de hilos.

MPI

Mejor para: Problemas que se pueden dividir en partes independientes, computación masivamente paralela, clusters de computadoras.

Limitaciones: Más complejo de programar, comunicación explícita requerida.

Protocolos de Coherencia de Caché

MSI (Modified, Shared, Invalid)

Modified: Línea modificada, solo en este caché
Shared: Línea válida, posiblemente en otros cachés
Invalid: Línea no válida

MESI (Modified, Exclusive, Shared, Invalid)

Modified: Línea modificada, solo en este caché
Exclusive: Línea válida, solo en este caché
Shared: Línea válida, en múltiples cachés
Invalid: Línea no válida

False Sharing

Problema:

Múltiples hilos acceden a variables diferentes en la misma línea de caché, causando invalidaciones innecesarias.

Ejemplo Problemático:

struct Data {
  int counter1; // Hilo 1 modifica
  int counter2; // Hilo 2 modifica
} data;

Solución - Padding:

struct Data {
  int counter1;
  char padding[64]; // Padding para separar líneas
  int counter2;
} data;

Técnicas de Optimización:
