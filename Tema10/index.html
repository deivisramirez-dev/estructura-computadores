Definición

"Sistema de comunicación: conjunto de componentes hardware o software que se encarga de la comunicación de datos entre computadores, en multicomputadores, y entre procesadores y memorias, en multiprocesadores."

Multiprocesadores UMA

Características:

  • Comunicación mediante acceso a memoria
  • Transparente al programador
  • Controladores de memoria como interfaz
  • Protocolo de coherencia de caché
  • Tamaño de paquete fijo (línea de caché)

Multicomputadores

Características:

  • Comunicación por paso de mensajes
  • Visible como subsistema de E/S
  • Control de tamaño de paquetes
  • Mayor probabilidad de errores
  • Asistente de comunicación

Topologías de Red Avanzadas

Arquitecturas de interconexión para sistemas de alto rendimiento

Hypercubes (n-Cubos)

Características:

  • N nodos conectados en un n-cubo
  • Grado de nodo: log₂(N)
  • Diámetro: log₂(N)
  • Bisection width: N/2

Ejemplo 3-Cubo (8 nodos):

001
010
011
100
101
110
111

Cada nodo conectado a 3 vecinos (diferencia de 1 bit)

Ventajas:

  • Diámetro pequeño
  • Escalabilidad logarítmica
  • Alto bisection width
  • Algoritmos de enrutamiento simples

Torus (Redes Toroidales)

Características:

  • Red 2D o 3D con conexiones wraparound
  • Grado de nodo: 4 (2D) o 6 (3D)
  • Diámetro: O(√N) para 2D
  • Bisection width: 2√N para 2D

Ventajas:

  • Grado de nodo constante
  • Buena escalabilidad
  • Simplicidad de implementación
  • Alto bisection width

Fat Trees (Árboles Gordos)

Características:

  • Estructura jerárquica de árbol
  • Ancho de banda creciente hacia la raíz
  • Grado de nodo variable
  • Diámetro: 2 log₂(N)

Ejemplo Fat Tree:

Ancho de banda aumenta hacia la raíz

Ventajas:

  • Alto bisection width
  • Escalabilidad excelente
  • Sin bloqueo de red
  • Ideal para centros de datos

Dragonfly

Características:

  • Topología de tres niveles
  • Grupos de nodos interconectados
  • Grado de nodo bajo
  • Diámetro: 3

Algoritmos de Enrutamiento

Dimension-Order Routing

Enrutamiento siguiendo dimensiones específicas

XY Routing: Primero X, luego Y
ZXY Routing: Primero Z, luego X, luego Y

Adaptive Routing

Enrutamiento que se adapta al estado de la red

Minimal: Solo rutas de longitud mínima
Non-minimal: Permite rutas más largas

Deadlock Avoidance

Prevención de interbloqueos en la red

Turn Model: Restricciones de giro
Virtual Channels: Canales virtuales
